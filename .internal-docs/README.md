# Internal Documentation

This directory contains internal documentation files that are not intended for the public repository.

## Purpose

Internal docs track:
- Audit reports
- Code quality assessments
- Project status snapshots
- Conversation summaries
- Deep analysis reports

## File Types

### Audit Reports
- `AUDIT_REPORT.md` - General audit report
- `DEEP_AUDIT_REPORT_YYYY-MM-DD.md` - Detailed audit reports
- `FULL_DEEP_AUDIT_YYYY-MM-DD.md` - Comprehensive audits

### Status & Tracking
- `PROJECT_STATUS.md` - Project status snapshot
- `CODE_QUALITY_TODO.md` - Code quality improvements
- `CONVERSATION_SUMMARY.md` - Development conversation summary

## Usage

These files are:
- **Not committed** to git (in `.gitignore`)
- **Local only** - for internal tracking
- **Optional** - can be deleted if not needed
- **Historical** - record of development process
